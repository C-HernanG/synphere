# Security Policy

## Reporting

Do not open public issues for vulnerabilities, secret exposure, or trust-and-safety bypasses.

Instead, report privately to the project maintainers through a dedicated security contact before public disclosure is enabled for the repository.

## Scope

Security reports may include:

- authentication or authorization bypasses
- secret leaks
- injection vulnerabilities
- data exposure risks
- room or moderation abuse vectors

## Repository Hygiene

- never commit secrets
- never commit real production exports
- rotate any credential immediately if it is exposed
- keep security-sensitive operational details out of the public repository
